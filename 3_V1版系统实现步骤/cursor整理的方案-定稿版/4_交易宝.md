# 交易宝产品方案

本文档描述交易宝模块的完整产品方案，包括界面功能、数据来源、计算公式、数据约束和补充说明。交易宝是交易员进行期货交易操作的核心模块，支持PC端和手机端两种使用场景。

---

## 一、界面功能描述

### 1.1 页面入口与权限

- **菜单路径**：一级菜单「交易宝」→ 二级菜单「交易终端」
- **权限要求**：交易员权限（主要使用者为交易员角色）
- **使用场景**：
  - PC端：完整的交易功能界面，采用 Grid 布局
  - 手机端：简化的交易功能界面（手机分辨率适配）

### 1.2 界面布局（PC端）

交易终端采用 Grid 布局，分为以下区域：

- **左侧指令列表区域**（3列宽，12行高）：显示指令列表，支持 Tab 切换
- **顶部账户资金区域**（9列宽，1行高）：账户选择器、保值方案选择器、资金信息
- **中间左侧盘口区域**（2列宽，4行高）：显示当前合约的盘口信息
- **中间左侧下单板区域**（2列宽，7行高）：下单表单
- **右侧自选合约区域**（7列宽，3行高）：自选合约列表
- **右侧数据列表区域**（7列宽，8行高）：委托/持仓/成交/平仓/资金 Tab 页签
- **底部状态栏**：连接状态、更新时间

### 1.3 左侧指令列表

#### 1.3.1 指令列表 Tab

**位置**：页面左侧（3列宽，12行高）

**Tab 切换**：

- **合同信息**：合同编号、客商、更新时间
- **交易信息**：合同类型（采购/销售）、合同数量（吨）、期货方向、期货数量（手）、期货合约
- **操作按钮**：
  - **认领**：认领/取消认领指令
  - **撤销处理**：同意/拒绝撤销请求
  - **完成**：完成指令
  - **备注**：添加备注

#### 1.3.3 指令详情抽屉

**显示内容**：

- **合同详情**：点价方式、点价窗口、基差、已点/未点、物流、仓库
- **子指令与敞口结构**：子指令列表（子指令ID、类型、状态、操作人、方向、影响手数）、指令日志时间线
- **备注与日志**：备注列表（作者、内容、时间）

**操作功能**：
- **认领/取消认领**：认领或取消认领指令
- **撤销处理**：处理撤销请求
- **完成指令**：完成指令
- **填入下单板**：将指令信息填充到下单板

### 1.4 顶部账户资金区域

#### 1.4.1 账户选择器

- **位置**：顶部左侧
- **功能**：下拉选择框，显示当前可用的期货账户列表
- **数据来源**：从「全局设置 → 期货账户管理」中读取，仅显示「启用且允许 Web 交易」的账户
- **显示规则**：
  - 如果账户设置了别名，则显示别名
  - 如果没有设置别名，则显示账户编码
- **切换行为**：切换账户后，页面自动刷新持仓、委托、成交、资金等数据

#### 1.4.2 保值方案选择器

- **位置**：账户选择器右侧（在下单板中）
- **功能**：下拉选择框，显示当前账户关联的保值方案列表
- **数据来源**：根据选择的期货账户，从「全局设置 → 期货账户管理」中读取关联的保值方案
- **显示规则**：仅显示启用状态的保值方案
- **切换行为**：切换保值方案后，影响指令执行和风控检查的范围

#### 1.4.3 资金信息显示

**位置**：账户选择器右侧

**显示内容**：
- **风险度**：风险度百分比，使用颜色区分（正常绿色，危险红色）
- **可用资金**：可用资金金额
- **持仓盈亏**：浮动盈亏金额，使用颜色区分（盈利红色，亏损绿色）
- **平仓盈亏**：已实现盈亏金额，使用颜色区分（盈利红色，亏损绿色）

**数据来源**：期现交易服务，实时同步

## 三、计算公式

### 3.1 浮动盈亏计算

**公式**：
```
浮动盈亏 = (最新价 - 持仓成本) × 持仓手数 × 合约乘数 × 方向系数
```

**方向系数**：
- 多头：+1
- 空头：-1

### 3.2 占用保证金计算

**公式**：
```
占用保证金 = 持仓手数 × 合约乘数 × 最新价 × 保证金比例
```

**数据来源**：
- 合约乘数：从合约信息中获取
- 保证金比例：从期货账户配置中获取

### 3.3 风险度计算

**公式**：
```
风险度 = (占用资金 / 总资产) × 100%
```

**风险等级**：
- 风险度 < 50%：正常（绿色）
- 50% ≤ 风险度 < 80%：警告（黄色）
- 风险度 ≥ 80%：危险（红色）

### 3.4 成交金额计算

**公式**：
```
成交金额 = 成交价格 × 成交手数 × 合约乘数
```

### 3.5 总资产计算

**公式**：
```
总资产 = 可用资金 + 占用资金 + 浮动盈亏
```

## 四、数据约束

### 4.1 下单约束

#### 4.1.1 交易时间约束

- **开市时间**：只能在交易时间内下单
- **收市时间**：收市后不能下单，但可以撤单
- **集合竞价**：集合竞价期间，部分下单功能受限

#### 4.1.2 资金约束

- **可用资金检查**：下单前检查可用资金是否充足
- **保证金检查**：下单前检查占用保证金是否超过账户限额
- **风险度检查**：下单前检查风险度是否超过阈值

#### 4.1.3 持仓约束

- **保值额度检查**：下单前检查是否超过保值方案配置的保值持仓额度
- **持仓方向检查**：平仓时检查持仓方向是否匹配
- **持仓数量检查**：平仓时检查持仓数量是否充足

#### 4.1.4 权限约束

- **账户权限**：只能操作有权限的期货账户
- **保值方案权限**：只能操作有权限的保值方案
- **指令权限**：执行指令时检查指令权限配置

### 4.2 撤单约束

- **挂单状态**：只能撤未成交或部分成交的委托单
- **已成交**：已全部成交的委托单不能撤单
- **已撤销**：已撤销的委托单不能重复撤单

### 4.3 策略约束

- **策略启用**：只有启用状态的策略才会执行
- **策略参数**：策略参数必须完整且合法
- **策略冲突**：同一合约的多个策略不能冲突

### 4.4 数据同步约束

- **实时性**：持仓、委托、成交、资金数据需要实时同步
- **一致性**：数据同步需要保证一致性，避免数据不一致
- **断线重连**：连接断开后需要自动重连，恢复数据同步

## 五、手机分辨率适配

### 5.1 分辨率判断

- **判断标准**：系统只判断一个分辨率界限来区分设备类型（参考 0_系统缺省默认.md）
- **分辨率界限**：默认 768px（可配置）
- **大于等于该界限**：使用 PC 端布局
- **小于该界限**：使用移动端布局

### 5.2 手机端一级目录显示

- **手机分辨率登录时，只提供以下一级目录**：
  - 工作台（指令工作台、通知列表）
  - 交易宝（需要做一个手机分辨率适配的交易宝）

### 5.3 手机端交易宝适配要求

#### 5.3.1 界面布局适配

**整体布局**：
- 采用单列垂直布局，避免横向滚动
- 顶部固定工具栏，包含账户选择、连接状态
- 主要内容区域可垂直滚动

**下单板适配**：
- 自动开平下单板和传统下单板合并为一个简化下单板
- 合约选择使用全屏下拉选择器
- 买卖方向使用大按钮，便于点击
- 数量输入使用数字键盘
- 三键下单简化为常用操作

**报价面板适配**：
- 报价面板采用卡片式布局
- 每个合约报价显示为独立卡片
- 支持左右滑动切换合约
- 关键信息（最新价、涨跌）突出显示

**持仓/委托/成交列表适配**：
- 列表采用卡片式布局，替代表格
- 每个记录显示为独立卡片
- 关键信息（合约、方向、数量、价格）突出显示
- 支持下拉刷新
- 支持上拉加载更多

**资金面板适配**：
- 资金信息采用卡片式布局
- 关键指标（可用资金、风险度）突出显示
- 使用大字体和颜色区分

#### 5.3.2 操作流程简化

**下单流程**：
- 简化下单步骤，减少操作层级
- 常用操作（如三键下单）一键完成
- 支持快速下单（预设数量、价格）

**查询流程**：
- 简化筛选条件，使用抽屉式筛选面板
- 支持快速筛选（常用条件）
- 支持搜索功能

**策略管理**：
- 手机端仅支持查看和启用/禁用策略
- 策略创建和编辑在 PC 端完成

#### 5.3.3 交互优化

**触摸优化**：
- 按钮尺寸不小于 44px × 44px
- 按钮间距足够，避免误触
- 支持手势操作（如滑动、长按）

**响应速度**：
- 优化加载速度，减少数据量
- 使用懒加载，按需加载数据
- 支持离线缓存

**错误提示**：
- 错误提示使用 Toast 或 Modal
- 提示信息简洁明了
- 支持快速重试

#### 5.3.4 样式规范

**字体大小**：
- 标题：18px - 20px
- 正文：14px - 16px
- 辅助信息：12px

**颜色规范**：
- 买入：红色（#ff4d4f）
- 卖出：绿色（#52c41a）
- 盈利：绿色（#52c41a）
- 亏损：红色（#ff4d4f）
- 正常状态：蓝色（#1677ff）
- 警告状态：橙色（#faad14）
- 危险状态：红色（#ff4d4f）

**间距规范**：
- 卡片内边距：12px - 16px
- 卡片间距：8px - 12px
- 按钮间距：8px

**圆角规范**：
- 卡片圆角：8px - 12px
- 按钮圆角：6px - 8px

#### 5.3.5 功能限制

**手机端不支持的功能**：
- 策略创建和编辑（仅支持查看和启用/禁用）
- 自组合报价创建（仅支持查看）
- 复杂筛选条件（仅支持常用筛选）
- 批量操作（仅支持单条操作）

**手机端特有功能**：
- 快速下单（预设常用参数）
- 语音输入（合约代码、数量等）
- 手势操作（滑动切换、长按菜单）

### 5.4 界面状态保存

- **保存内容**：用户当前打开的页面、页面布局配置、筛选条件等
- **保存方式**：将用户当前页面和布局信息传到期现服务端进行记录
- **设备类型区分**：PC 端和移动端（手机）的界面状态需要分开存储
- **异地登录恢复**：用户在不同地点登录时，系统会加载该用户在该设备类型下保存的界面状态

## 六、其他补充

### 6.1 连接管理

#### 6.1.1 交易连接

- **连接方式**：WebSocket 长连接
- **心跳机制**：定期发送心跳包，保持连接活跃
- **断线重连**：检测到断线后自动重连，最多重试 3 次

- **断线重连**：检测到断线后自动重连
- **数据缓存**：断线期间缓存最新行情数据

### 6.2 数据刷新

#### 6.2.1 自动刷新

- **持仓数据**：实时推送更新
- **委托数据**：实时推送更新
- **成交数据**：实时推送更新
- **资金数据**：实时推送更新
- **行情数据**：实时推送更新

#### 6.2.2 手动刷新

- **刷新按钮**：提供手动刷新按钮
- **下拉刷新**：手机端支持下拉刷新
- **刷新提示**：刷新过程中显示加载状态

### 6.3 错误处理

#### 6.3.1 下单错误

- **错误提示**：下单失败时显示具体错误信息
- **错误分类**：资金不足、持仓不足、权限不足、交易时间错误等
- **错误处理**：根据错误类型提供相应的处理建议

#### 6.3.2 连接错误

- **错误提示**：连接失败时显示错误信息
- **重连机制**：自动尝试重连
- **降级方案**：连接失败时提供离线模式（仅查看历史数据）

### 6.4 性能优化

#### 6.4.1 数据优化

- **分页加载**：列表数据分页加载，避免一次性加载大量数据
- **懒加载**：按需加载数据，减少初始加载时间
- **数据缓存**：缓存常用数据，减少请求次数

#### 6.4.2 渲染优化

- **虚拟滚动**：长列表使用虚拟滚动，提高渲染性能
- **防抖节流**：输入框和滚动事件使用防抖节流，减少计算次数
- **组件懒加载**：按需加载组件，减少初始包大小

### 6.5 安全要求

#### 6.5.1 权限控制

- **账户权限**：只能操作有权限的期货账户
- **操作权限**：下单、撤单等操作需要相应权限
- **数据权限**：只能查看有权限的数据

#### 6.5.2 操作确认

- **重要操作确认**：下单、撤单等重要操作需要二次确认
- **风险提示**：高风险操作（如大额下单）需要风险提示
- **操作日志**：记录所有操作日志，支持审计

## 七、与其他模块的关联

### 7.1 与指令宝的关联

- **指令执行**：交易宝从指令宝获取待执行的指令
- **执行结果**：交易宝执行指令后，将执行结果回传给指令宝
- **指令状态**：指令执行状态实时同步到指令宝

### 7.2 与匹配宝的关联

- **委托单数据**：交易宝生成的委托单同步到匹配宝
- **成交数据**：交易宝的成交数据同步到匹配宝
- **期现关联**：匹配宝可以关联交易宝的委托单和合同

### 7.3 与风控宝的关联

- **风控检查**：交易宝下单前调用风控宝进行风控检查
- **持仓监控**：风控宝监控交易宝的持仓情况
- **风险预警**：风控宝发现风险时，向交易宝发送预警信息

### 7.4 与全局设置的关联

- **账户配置**：从全局设置读取期货账户配置
- **保值方案**：从全局设置读取保值方案配置
- **权限配置**：从全局设置读取权限配置

## 八、开发注意事项

### 8.1 实时性要求

- **数据同步**：持仓、委托、成交、资金数据需要实时同步，延迟不超过 1 秒
- **行情推送**：行情数据需要实时推送，延迟不超过 500 毫秒
- **状态更新**：连接状态、操作状态需要实时更新

### 8.2 稳定性要求

- **断线重连**：连接断开后需要自动重连，保证服务可用性
- **错误恢复**：操作失败后需要提供错误恢复机制
- **数据一致性**：保证数据的一致性，避免数据不一致

### 8.3 用户体验要求

- **响应速度**：操作响应时间不超过 500 毫秒
- **加载速度**：页面初始加载时间不超过 2 秒
- **错误提示**：错误提示清晰明了，提供处理建议

### 8.4 兼容性要求

- **浏览器兼容**：支持 Chrome、Firefox、Safari、Edge 等主流浏览器
- **移动端兼容**：支持 iOS、Android 等主流移动操作系统
- **分辨率适配**：支持不同分辨率的 PC 和移动设备
