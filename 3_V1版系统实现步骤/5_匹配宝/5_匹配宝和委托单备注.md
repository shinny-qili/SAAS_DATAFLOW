目录路径：
    一级：匹配宝
    二级：匹配宝与委托备注

权限要求：
    匹配宝页面操作必须有匹配员权限

一个委托单列表
    筛选
        当日 or 历史（单选）
    按钮
        期现备注（需选中行）
        委托备注（需选中行）
        批量匹配（需选中行）
        单条匹配（需选中行）
        导出
        刷新
        搜索
    表格列（一级字段）
        单选radio_button（行选择）
        期货账户
        保值方案
        日期时间
        短编号
        合约
        方向（买入/卖出）
        成交均价
        成交手数
        期现备注（表中表形式展示，包含二级字段）
        报单价格
        未匹配手数
        挂单状态（未成/部成/全成/已撤/部成已撤/错单）
        委托备注（为空时显示空字符串）
    表格列（期现备注二级字段，纳入表格列设置）
        手数
        现货数量
        点价价格
        标签
        备注
        客商
        合同号
        是否已匹配（在表格中展示：是/否）
    分页
        与其他页面一致（默认每页10条，支持10/20/50/100）
    表格列设置
        支持自定义列显示和列宽
        期现备注的二级字段可单独控制显示/隐藏

期现备注功能
    UI形式
        弹窗编辑
    数据关系
        一条期货可以附加多条期现备注
    字段列表
        手数（必填，数字）
        现货数量（数字）
        点价价格（数字）
        标签（下拉选择，来源：保值方案设置的标签）
        备注（文本）
        客商（下拉选择，来源：客商列表，如果已选合同则不能选）
        合同号（下拉选择，来源：同保值方案的合同列表，如果保值方案是单单才允许选取）
        是否已匹配（布尔值）
    表格展示
        期现备注以表中表形式展示
        二级字段直接展示在表格中，支持在列设置中控制显示/隐藏
    操作功能
        支持新增期现备注
        支持编辑期现备注（每一行）
        支持删除期现备注（每一行，无需确认对话框）
    删除规则
        整个系统都不做弹框内在弹框，删除操作直接执行，无需确认对话框
    数据验证
        期现备注中的总手数不能大于委托单的已成交手数
        如果保值方案是单单，才允许选取合同，合同数量和价格非必填
    期现备注接口
        期货id：int
        期现备注列表：json_list
            [
                {
                    手数：int,
                    现货数量：float,
                    点价价格：float,
                    标签：str,
                    备注：str,
                    客商id：int（可选，如果已选合同则不能选）,
                    合同id：int（可选，如果保值方案是单单才允许选取）,
                    是否已匹配：bool
                }
            ]

委托备注功能
    UI形式
        弹窗编辑，支持富文本
    字段
        期货id：int
        备注信息：str（富文本）
    表格展示
        完整显示，显示不下时附加"……"
        鼠标悬浮时在泡泡框中显示完整内容
    操作功能
        支持编辑委托备注
        支持删除委托备注（无需确认对话框）
    删除规则
        整个系统都不做弹框内在弹框，删除操作直接执行，无需确认对话框
    委托备注接口
        期货id：int
        备注信息：str（富文本）

批量匹配功能
    UI形式
        弹窗显示
    操作流程
        1. 先选择期货委托（可多选）
        2. 点击批量匹配按钮
        3. 弹出批量匹配弹窗
        4. 弹窗中显示分配确认表（可编辑）
        5. 区分符合匹配和不符合匹配的内容
        6. 确认时只将符合匹配规则的内容完成匹配
        7. 支持一次匹配多条记录
        8. 点击确认匹配后，再弹一个窗显示匹配结果
    弹窗内容
        分配确认表（可编辑），分为两部分：
        期货信息：
            保值方案
            期货账户
            日期时间
            合约
            方向
            开平
            价格
            手数
            备注
            客商
        现货信息：
            合同号
            标签
            点价价格
            现货数量
    匹配规则
        如果保值方案是单单，且标签是以下之一，必须指定合同：
            长单保值
            长单增量
            长单减量
            长单取消
            订单接货保值
            订单出货保值
            订单增量
            订单减量
            订单取消
            买点价
            卖点价
            调敞口
            代持
        其他情况：绑定保值方案进行匹配
    批量匹配接口
        匹配记录列表：json_list
            [
                {
                    保值方案id：int,
                    合同id：int（可选，根据匹配规则决定）,
                    合同数量：float（可选，非必填）,
                    合同价格：float（可选，非必填）,
                    期货id：int,
                    期货数量：int
                }
            ]

单条匹配功能
    UI形式
        弹窗确认
    操作流程
        1. 选择一条期货委托
        2. 点击单条匹配按钮
        3. 弹出匹配确认弹窗
    匹配逻辑
        如果保值方案是整体、客商、人员的：
            只需要确认即可完成匹配
        如果保值方案是单单的：
            需要选择指定合同
    匹配成功后
        自动创建期现备注
        刷新委托单列表
    单条匹配接口
        保值方案id：int
        期货id：int
        合同id：int（可选，如果保值方案是单单则必填）
        合同数量：float（可选，非必填）
        合同价格：float（可选，非必填）
        期货数量：int

数据来源说明
    标签选项来源
        来自保值方案设置的标签
        每笔期货都应该有保值方案
    合同选项来源
        来自同保值方案的合同列表
    客商选项来源
        来自客商列表
        如果已选合同，则不能选客商
