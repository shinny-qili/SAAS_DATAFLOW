定位与终端形态
    浏览器运行，默认嵌入公司统一工作台，也可以独立窗口。
    目标体验靠近快期2，支持键盘快捷键、深色/浅色主题、布局自定义。
    通过HTTPS + WebSocket连接期现交易服务，支持断线提示与自动重连。

交互与能力
    布局联动：
        左侧指令卡片、盘口数据、自选列表、下单板、右侧列表按照12栅格布局自适应
        盘口数据与下单板/自选单击保持同一合约
    账户切换：
        切换即触发重新拉取账户快照、资金、行情订阅
        保留当前布局、自选分组以及指令筛选条件
    保值方案切换：
        实时重新筛选持仓/委托/期现备注
        下单板默认联动选中的保值方案，可手动修改
    指令卡片与抽屉：
        卡片支持按标签页过滤、拖拽改变优先级（可选）
        行内“认领/完成/匹配”等操作即时反馈到列表
        点击卡片打开抽屉，抽屉中的操作会回写到卡片和下单板
        “一键生成下单参数”会同步指令编号、保值方案、合约、买卖、数量、价格到下单板
    盘口与行情订阅：
        前端通过WebSocket订阅当前所关注的合约
        自选切换、下单板合约修改、指令填充都会触发盘口刷新
        行情字段按照涨跌红绿规则自动变色
        自选列表：
            输入框新增/删除合约
            标签页自选1/2/3与外盘行情分组
            支持拖拽排序
            单击填充到下单板
    下单板：
        支持合约手动输入、自选点击、指令填充三种来源
        数量、价格提供增减按钮和键盘输入，支持自动取指令参数
        标签下拉根据当前保值方案读取自定义标签，为单选，用于驱动匹配宝期现备注
        价格类型（对价/挂价/市价/指定）决定价格输入逻辑
        开平、套保由单选/复选控制，保证金与手续费实时计算
        下单成功/失败以弹窗+状态条形式提示
    右侧数据列表：
        委托/持仓/成交/平仓四个列表与下单板、指令联动
        委托列表支持“填单到下单板”“撤单”等快速操作
        列表支持状态筛选、关键字搜索、导出
    状态条与连接：
        底部状态条实时显示行情连接、交易连接、最新时间戳
        断线时自动置灰相关操作并提示
    快捷键：
        Esc：关闭抽屉或弹窗

    指令详情抽屉（侧滑面板）
        触发方式：点击左侧指令列表中的指令行
        功能定位：显示指令详细信息，执行指令相关操作
        标签页：我发起的、我认领的、全部（与左侧列表同步）
        分区结构：
            1）基础信息区（卡片）：
                指令类型、状态、指令编号、保值方案、发起人、创建时间/更新时间、关联合同编号、交易员类型（点价/均价/一口价/投机等）
            2）合同详情区（可折叠）：
                点价类：合同号、点价方式、点价窗口、基差、已点/未点数量、价格限制、交付方式、交期、仓库、关联现货条款
                均价类：当日/当月/历史均价需求、各合同分摊明细、均价期、净值敞口、均价进度
                一口价/采销：合同条款、计价方式、交付方式、合同数量及历次增减记录
                其他类型：根据创建类型展示“创建合同信息补充”“变更合同信息补充”“撤单原因”等
            3）子指令&敞口结构（树形+表格）：
                父子结构：展示父指令与子指令树，可展开查看每条子指令状态（待执行、已完成、已拒绝等）
                每条子指令字段：类型、创建人、状态、影响敞口（方向/手数）、完成时间、执行人
                敞口结构可视化：显示来源（合同、投机、移仓等）、占比、已匹配/未匹配数量
                敞口变更明细：记录每次子指令执行对敞口的增减过程
            4）备注与日志：
                相关人备注：按时间轴展示所有参与人填写的备注，可筛选人员/角色
                指令更新日志：记录合同信息变更、数量增减、撤销申请等，说明更新原因
                执行日志：展示认领、下单、完成、拒绝等每一步操作及操作人、时间
            5）关联操作区：
                显示与该指令关联的匹配宝操作、委托记录、成交记录，可点击跳转
        操作按钮：
            认领/取消认领
            同意/拒绝（有撤销申请时显示）
            完成指令
            手工匹配期货
            完成期货交易
            备注
            生成下单参数（自动填充到下单板）
        执行信息：
            执行进度显示（整体进度+子指令进度）
            执行日志展示
            关联的委托单、成交单信息

风控与提示
    柜台/行情断线时，锁定下单按钮，仅保留"撤单"。
    底部状态栏实时显示账户资金和盈亏信息。

数据同步
    WebSocket推送：行情、委托、成交、连接状态。
    HTTP轮询：列表初始加载、历史查询（分页）。
    服务端布局自动存储：保存布局、主题、快捷键设置、自选合约列表。

集成接口（V1）
    GET /api/trade/web/settings            → 加载布局、快捷键、模板
    GET /api/trade/accounts                → 可用账户+状态
    GET /api/trade/hedge-schemes           → 当前交易员支持的保值方案
    POST /api/trade/orders                 → 下单
    POST /api/trade/orders/{id}/cancel     → 撤单
    GET /api/trade/orders                  → 委托列表
    GET /api/trade/fills                   → 成交列表
    GET /api/trade/positions               → 持仓列表
    GET /api/trade/closed                  → 平仓列表
    GET /api/trade/funds                   → 资金信息

测试要点
    账户切换后1s内完成行情与数据刷新。
    自动开平下单板在键盘模式下不能触发重复提交。
    指令执行抽屉的回写结果与交易服务事件一致。

实现说明（V1原型）
    界面实现：仅实现界面布局和静态数据展示，不实现WebSocket和快捷键功能
    数据来源：
        交易账户：从期货账户管理获取
        保值方案：从保值方案管理获取，筛选当前交易员支持的方案
        指令数据：从指令工作台获取
        持仓/委托/成交/资金：使用模拟静态数据
    功能限制：
        不实现WebSocket实时推送，使用静态数据
        不实现快捷键功能
        不实现主题切换
        下单操作仅做界面交互，不实际提交
        指令执行抽屉仅展示指令列表，不实现实际执行逻辑
