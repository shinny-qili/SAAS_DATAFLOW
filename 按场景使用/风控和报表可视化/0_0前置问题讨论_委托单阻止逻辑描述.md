事中判断提醒和阻止交易
    风控形态
        规则结构：指标(左侧) + 运算符 + 阈值 + 时间窗口 + 适用范围(账户/品种/到期月/交易员/法人与分部) + 处置方式(提示/二次确认/强拦截/提交审批) + 豁免(白名单/临时豁免时长)。
        触发方式：事件驱动（下单/改单/撤单/成交/保证金变更/行情快市）+ 定时轮询（秒级）。
        审计：所有命中、拦截、放行、人工豁免均留痕（人、时、规则、原始值、阈值、证据快照）。
    风控维度
        钱可用的风控维度如下：
            账户可用资金、总权益、占用保证金、预占用保证金
        仓可用的风控维度如下：
            净/单边仓位上限（品种/合约/到期月/交易员）
        单可用的风控维度如下：
            单笔手数上限、单日累计手数、日内委托笔数
        行情与流动性可用的风控维度如下：
            涨速/波动率阈值、价差与基差异常
            流动性阈值（盘口累积数量低于阈值）
        敞口可用的风控维度如下：
            净暴露上限（名义/Delta），覆盖率目标区间与偏离阈值
            单客商/单合同映射后的敞口限额
    风控结果
        阻止开仓
        需审批豁免
报表
    事后
        战略视角
            现货敞口-对冲头寸-现金及保证金-利润波动
                敞口暴露和套保覆盖率
                资金占用与场景损益
                极端风险监控：历史高波动区间回放、最大回撤、保证金占用
        腰部视角
            保值方案:持仓结果（根据成交记录计算），开平仓，利润，成本（手续费，保证金占用）
            管理口径：期货平仓和浮动盈亏、基差贡献分布、手续费、资金占用机会成本
            期货账户：钱，仓，委托，开平仓，成本（手续费，保证金占用），交割
            合同：交割、点价、盈亏、基差
            虚拟库存：基差、构成要素
        底层缺失报警
            勾选“保值优先”后缺少对应月份/合约映射。
            合同缺少定价条款/交期/升贴水参数。
            品级/地区与期货标的映射缺失或过期。
            行情/清算/库存数据中断与延迟。
    事前
        降低行情极端风险
            VaR（历史模拟/蒙特卡洛/参数法可选）、压力测试场景库（历史极端日、跨市场联动、基差骤变、汇率冲击）。
            波动率曲线与涨速监控：σ期限结构、ATR与价差分位点。
        保证金预警
            通过交易日历，节假日/交割，提醒提保风险
            设置保证金阈值、行情涨速对于保证金压力，提前警示超额风险
            利用率优化：历史保证金占用/权益利用率对比，提示闲置资金与建议（移仓降占用、合约切换）。
        因子分析
            把行情波动分解为现货、时政等因素
