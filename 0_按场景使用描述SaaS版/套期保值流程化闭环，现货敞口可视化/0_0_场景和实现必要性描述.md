套期保值流程化闭环场景，产业客户
    需要在期现业务中有一条作为基础的，高自动化的标准操作流程
    需要知道现货敞口结果和敞口来源，并针对敞口进行后续期货操作
    在操作流程中需要审批插入
    对于操作结果有可视化方案

期现系统有一个流程化闭环有以下优点
    利于推广，客户领导很喜欢标准化这一套跑通
    作为系统的脊柱并延申功能，避免盲目扩充系统功能
    充分从这个场景考虑易用性和实用性，场景和方案可以控制在适当范围，利于实施

潜在问题
    暂时只是小闭环，如果客户需要财务凭证，需要扩充闭环范围
    更精细的敞口计算，比如在途距离、天气、外汇等，需要扩充敞口计算规则
